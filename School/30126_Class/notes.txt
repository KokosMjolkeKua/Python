DocStrings & Type Hints

Docstring:
- text to help understand what goes on in the code
- use it forward to show your thinking process
- it should be possible to understand the function based on the DocStrings(for any human)

Type-Hints:
- you can tell python to put their sight on one datatype more than another
- basically a hint to make sure there are no errors(syntax or otherwise)

Logging:
- writing what you are doiing
- what the game is supposed to do
- log information somewhere that you can access later, a library
- stdout works well(terminal), as long as you are in the same environment
- with logging you choose where the info end up 
    levels, file output, timestamps, ddebug, info , warning, error, criical
    proffessional apps use logging everywhere
